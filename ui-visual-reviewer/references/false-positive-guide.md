# False Positive Guide

`getComputedStyle` ベースの監査で発生しやすい偽陽性と対処法。

## 1. アニメーション中間状態

**症状**: `opacityStack×0.38` のような中途半端な値。コントラスト 1.x:1。

**原因**: Puppeteer がスクリプトを実行したタイミングで、opacity のCSS transitionが途中だった。

**確認**: スクリーンショットを目視。実際にテキストが読めるなら偽陽性。

**対処**: `page.screenshot()` 前に `await sleep(1000)` を追加してアニメーション完了を待つ。

## 2. プレス状態オーバーレイ

**症状**: ボタンのテキストが bg とほぼ同色（1.0〜1.5:1）として報告される。

**原因**: React Native Web の `Pressable` がプレス後に `opacity` オーバーレイを残すことがある。
自動化クリック後の状態で監査すると、このオーバーレイ要素を拾うことがある。

**例（countstack）**: 「メンバー追加」ボタン → fg `#10182e` on bg `#111318` = 1.05:1 と報告される。
実際はボタンテキストは `textPrimary (#E3E2E6)` で読める。

**確認**: スクリーンショットで目視。ボタンが読めるなら偽陽性。

**対処**:
1. クリック後に別の中立的な要素をクリックしてプレス状態をリセット
2. または: クリック操作の前後で監査する（クリック前なら偽陽性が出ない）

## 3. disabled 要素

**症状**: `textDisabled (#74777f)` on surface → 3.76:1 などAA未達として報告される。

**原因**: disabled状態のテキストはWCAGの適用外だが、スクリプトは区別しない。

**判断**: disabled であることが明確なら、ユーザービリティの観点でも問題なければ容認。
ただし「disabled なのに読めない」は UX問題として記録する価値がある。

**countstack の例**: 「直接入力」ラベル 3.76:1 → モーダルのプレースホルダーラベルで disabled 扱い。
実際には thin/small テキストで読みにくい可能性があり、要検討。

## 4. SVG fill / stroke

**症状**: SVG アイコンの色が取得されない・bg `rgb(0,0,0)` と報告される。

**原因**: `getComputedStyle(el).color` は SVG の `fill` を返さない。

**対処**: SVG 要素は手動で `fill` の値を確認する。

## 5. position:absolute オーバーレイ

**症状**: 実際の要素ではない装飾用オーバーレイが要素として拾われる。

**原因**: `pointer-events: none` の絶対配置要素が `textContent` を持つ場合、`isAuditable` フィルタを通る。

**確認**: `rect` の位置がおかしい（他の要素と完全に重なっている）ものを疑う。

## フィルタリングの指針

以下の条件のどれか1つでも当てはまる場合は手動確認に回す：

- `opacityStack < 0.5`
- コントラスト比 < 1.5:1（disabled でも同色は設計ミスの可能性）
- `matchedToken.token === null`（トークン外の色）
