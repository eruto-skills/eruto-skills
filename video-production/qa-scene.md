# QA Scene — Remotion シーン品質チェック

## 概要

カスタムアニメーションシーン（src/scenes/*.tsx）の品質を、スチル画像の自動生成+目視確認+警告分析で検証する。空画面・字幕エリア侵入・フェーズ切り替え不足を検出し、修正方針を提案する。

## 引数

- シーンID（省略時: 全シーン）: `s1-02`, `s1-02 s2-4`, `s1`（セクション単位）, `all`
- `--fix`: 検出した問題を自動修正して再検証する

例:
- `/qa-scene` → 全シーンをチェック
- `/qa-scene s1-02 s2-4` → 指定シーンのみ
- `/qa-scene --fix` → 全シーンをチェックして問題を自動修正

## 実行手順

### Step 1: スチル生成

```bash
cd projects/llm-skill-workflow
npm run qa -- [scene-ids]
```

出力先: `out/stills/qa/{scene-id}-{a,b,c,d}.png`
チェックポイント: a(30-35%), b(55%), c(75-80%), d(93%)

### Step 2: 警告の分類

QAスクリプトの出力を解析し、警告を以下の3カテゴリに分類する。

#### 要修正（Must Fix）

| 警告 | 条件 | 理由 |
|------|------|------|
| 空画面 | b(55%) or c(75%) チェックポイント | シーン中盤〜後半でコンテンツが足りない |
| 字幕エリア侵入 | > 3% | 字幕と重なって読めなくなる |

#### 許容（Acceptable）

| 警告 | 条件 | 理由 |
|------|------|------|
| 空画面 | a(30-35%) チェックポイント | アニメーション序盤で要素が出揃っていないのは自然 |
| 差分不足 | c→d 遷移 | 75%時点で全要素が表示済みなら93%と同じ画面で問題ない |
| 差分不足 | a→b 遷移 (diff > 0.5%) | フェーズの段階表示で差分が小さいだけの場合がある |

### Step 3: 目視確認

自動メトリクスだけで判断しない。**要修正**に分類された警告は、必ずReadツールでスチル画像を開いて目視確認する。

確認観点（`.claude/rules/remotion-qa.md` のチェックリスト）:
1. 画面占有率: コンテンツが画面の60%以上を使っているか
2. フォントサイズ: キーワード64px以上、補足32px以上
3. 枠の水増し: 大きな枠の中に小さいテキストがないか
4. 字幕エリア: 下部120pxにコンテンツが被っていないか
5. スマホ可読性: 28px未満のフォントがないか
6. 情報の空白時間: 主要素が3秒以内に表示開始するか
7. 要素の被り: テキストやパネルが重なっていないか

### Step 4: 修正提案（または自動修正）

要修正と判定した項目について、シーンのソースコード（src/scenes/*.tsx）を読み、具体的な修正を提案する。

#### よくある修正パターン

| 問題 | 修正方針 | 具体例 |
|------|---------|--------|
| 空画面（中盤） | アニメーション遅延を短縮 | `slideUp(18.5 + i * 2.5)` → `slideUp(18.5 + i * 1.2)` |
| 空画面（中盤） | フォントサイズ拡大 | `fontSize: 32` → `fontSize: 48` |
| 空画面（後半） | 要素追加 or 要素の表示開始を前倒し | `cardsAppear = 7` → `cardsAppear = 4.5` |
| スカスカ | ボックスサイズ拡大 | `width: 400` → `width: 560` |
| 字幕エリア侵入 | padding-bottom拡大 or 要素を上に移動 | `padding: "48px 80px 100px"` → `"48px 80px 140px"` |
| フォント小さい | fontSize引き上げ | 最低28px、キーワードは64px以上 |

`--fix` フラグがある場合:
1. 修正をEditツールで適用する
2. 修正したシーンだけ再度QAスチルを生成する
3. Readツールで改善を確認する
4. 改善されなければさらに調整する

### Step 5: レポート

日本語で結果を報告する。

```
## QA結果

チェック: 28シーン / 109スチル
要修正: 3件
許容: 5件
合格: 残り全て

### 要修正
- s1-02-b (55%): 空画面 — カード表示遅延が大きい。slideUp間隔を短縮推奨
- s2-4-d (93%): スカスカ — Phase3のフォント拡大推奨

### 許容（対応不要）
- s1-05-a (25%): 空画面 — 収束アニメーション途中、意図通り
- s3-04 c→d: 差分不足 — 75%で全要素表示済み
```

## デザイン基準の参照先

詳細なルールはプロジェクト内に定義済み:
- `.claude/rules/scene-layout.md` — レイアウト・タイポグラフィ・色・アニメーションの基準
- `.claude/rules/remotion-qa.md` — QAチェックリスト・フォントサイズ基準
- `knowledge/motion-graphics-design-guidelines.md` — モーショングラフィックスの設計指針

修正時はこれらのルールに準拠すること。特に:
- 1080pでの最小フォントサイズ: 28px
- キーワード: 64px以上
- 数字ヒーロー: 120-144px
- 字幕セーフゾーン: 下部120px（padding-bottom 140px推奨）
- アクセントカラー: 1シーン最大2-3色

## 委譲での使い方

このスキルは単体で呼び出すだけでなく、シーン修正ワークフローの一部として組み込める。

典型的な流れ:
1. シーンファイルを編集する
2. **このスキルを実行して品質を確認する**（← ここが自動化ポイント）
3. 問題があれば修正→再確認のサイクル
4. 問題なければコミット

Agentツールで並列実行する場合:
- 複数シーンの修正後、シーンIDを指定してQAを一括実行できる
- 修正とQAを別エージェントに分けることも可能
