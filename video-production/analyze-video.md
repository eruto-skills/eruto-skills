# Analyze Video — 参考動画の構造・テクニック分析

## 概要

YouTube動画のナレーション全文と構造を分析し、自分の動画制作に活かせるテクニックを抽出する。字幕（自動生成含む）からナレーション全文を復元し、「なぜこの動画が良いのか」を言語化して再現可能な知見に変換する。

## 引数

- YouTube URL（必須）
- `--compare <対象>`: 自分の動画や別の参考動画と比較する。対象はYouTube URLまたはプロジェクトパス（例: `llm-skill-workflow`）
- `--quick`: 簡易分析（構成とテクニック抽出のみ、詳細スコア省略）

## 実行手順

### Step 1: メタデータ・字幕の取得

yt-dlpでメタデータと字幕を取得する。Windows環境のため `PYTHONUTF8=1` を付ける。

```bash
# メタデータ取得（JSON）
PYTHONUTF8=1 yt-dlp -j "<URL>" > /tmp/yt_meta.json

# メタデータ読み取り（Python経由でエンコーディング問題を回避）
PYTHONUTF8=1 py -c "
import json
with open('/tmp/yt_meta.json', encoding='utf-8') as f:
    d = json.load(f)
for k in ['title','channel','duration_string','upload_date','description']:
    print(f'{k}: {d.get(k,\"N/A\")}')"
```

yt-dlpの `--print` はWindows環境でエンコーディングが壊れるため、必ずJSON出力 → Python解析の2段階で処理する。

```bash
# 日本語自動字幕のダウンロード
PYTHONUTF8=1 yt-dlp --write-auto-sub --sub-lang ja --sub-format vtt --skip-download \
  -o "C:/@projects/vids/ref_%(id)s" "<URL>"
```

字幕が取得できない場合（字幕なし動画）:
- `--sub-lang en` で英語字幕を試す
- それも無ければ、メタデータ（タイトル・説明文）のみで分析可能な範囲を報告し、ユーザーに判断を仰ぐ

### Step 2: 字幕のクリーンアップ

```bash
PYTHONUTF8=1 py C:/@projects/vids/scripts/clean-vtt.py ref_<ID>.ja.vtt ref_<ID>.txt
```

クリーンアップ後のテキストを全文読み込む。長い動画でも全文読み込みは必須。部分的な分析は精度が落ちるため行わない。

自動生成字幕には誤認識が含まれる。固有名詞の誤認識は動画の説明文やコンテキストから推定して補正する（例: 「テレトワールドリティクス」→「テレ東ワールドポリティクス」）。

### Step 3: 構造分析

ナレーション全文を読み、以下の構造を特定する。

#### 3a. マクロ構造（全体の骨格）

動画全体を大きなブロックに分割し、各ブロックの役割を特定する:

- **導入部**: 冒頭何分で何をしているか（挨拶、前提共有、問題提起、フック）
- **本論**: いくつのトピックに分かれているか、各トピックの長さと関係性
- **転換点**: 話題が切り替わる瞬間、その接続方法
- **結論部**: まとめ方、次回予告、CTA（Call to Action）の有無

構造を時系列で図示する:
```
[0:00-2:30] 導入: 挨拶+前提共有+今回の問い提示
[2:30-15:00] 本論1: ...
[15:00-18:00] ブリッジ: ...
[18:00-35:00] 本論2: ...
[35:00-43:00] 結論: ...
```

#### 3b. ミクロ構造（段落レベルのパターン）

繰り返し使われている説明パターンを抽出する:
- 主張→根拠→具体例
- 問いかけ→展開→結論
- 「一方で」「しかし」等の転換パターン
- 前提の積み上げ方（AだからB、BだからC...）

### Step 4: テクニック分析

以下の7軸で具体的なテクニックを抽出する。各軸で「この動画がやっていること」と「なぜそれが効果的か」を明記する。抽象的な評価（「良い」「効果的」）ではなく、ナレーションから具体的な引用を含めて示す。

#### 軸1: フック・引き込み

冒頭で視聴者を引き込む手法。最初の30秒〜1分を精密に分析する。

着目点:
- 最初の一文は何か（問いかけ？宣言？データ？）
- 「なぜ今この話が重要か」をどう伝えているか
- 視聴者の不安・関心・好奇心のどれに訴えているか
- 冒頭で提示される「問い」の射程（動画全体をカバーしているか）

#### 軸2: 情報密度とペーシング

単位時間あたりの情報量の配分。

着目点:
- 密度が高い区間と低い区間の交互パターン
- 「息継ぎ」の手法（具体例、繰り返し、要約）
- 早口で大量の情報を詰め込む区間があるか、その効果は何か
- 重要度に応じた尺の配分（重要な話に長い時間をかけているか）

#### 軸3: ヘッジ表現・炎上対策

断定を避ける表現、予防線、免責表現のパターン。

着目点:
- 「あくまで」「もちろん」「一概には言えない」等の頻出ヘッジ表現を列挙
- ヘッジの使い分け（どの主張にヘッジをかけ、どこは断定するか）
- 反論を先回りで潰す手法（「〜と思う方もいるかもしれませんが」）
- 情報源の明示（「〜によると」「報道によれば」）で責任を分散する手法
- 「個人の意見」と「事実」の境界をどう引いているか

この軸は特に重要。センシティブなテーマを扱う動画ほど、ヘッジの技術が洗練されている。具体的な表現を網羅的に拾い出すこと。

#### 軸4: 説得力・信頼性構築

視聴者に「この人の話は信頼できる」と思わせる手法。

着目点:
- 専門知識の提示方法（さりげなく？明示的に？）
- データ・出典の引用パターン
- 「私が調べた限り」等の個人的な調査努力の表現
- 複数の視点を提示してからの意見表明
- 限界の正直な提示（「ここは分からない」「情報が限られている」）

#### 軸5: エンゲージメント維持

43分など長い動画で視聴者を飽きさせない工夫。

着目点:
- トピック転換の頻度とタイミング
- 問いかけ・修辞的質問の挿入頻度
- 具体例・エピソードの配置
- テンションの上げ下げ（声のトーンは字幕から推測しにくいが、文体の変化は追える）
- 「次にお話しするのは」等の予告による期待感の醸成

#### 軸6: 記憶に残るフレーズ

視聴後も頭に残る表現、引用したくなる一文。

着目点:
- 対比・対句構造（「AではなくB」「AであるがBでもある」）
- 数字を使った印象的な表現
- 比喩・アナロジー
- キーワードの反復（同じ言葉を何度も使って刷り込む手法）
- 結論を一文に凝縮したフレーズ

#### 軸7: 自分の動画への応用

上記6軸の分析を踏まえ、自分の動画制作に具体的に活かせるテクニックを抽出する。

着目点:
- 自分のジャンル（技術解説）でも使えるテクニックはどれか
- 動画の長さの違い（参考: 43分 vs 自分: 13分）を踏まえた適用方法
- すぐに取り入れられるもの vs 段階的に習得すべきもの
- テクニックの具体的な適用例（「次の動画の冒頭で〜する」レベルの具体性）

### Step 5: 比較分析（--compare 指定時）

比較対象がYouTube URLの場合:
- 同じStep 1-4を比較対象にも実行する
- 両者の分析結果を軸ごとに並べて比較する

比較対象がプロジェクトパスの場合:
- `projects/<name>/generate-audio.sh` からナレーション原稿を読み込む
- `/review-content` の評価軸と本スキルの分析軸を対応させて比較する

比較レポートの構造:
```
## 比較: [参考動画] vs [自分の動画]

| 観点 | 参考動画 | 自分の動画 | 差分・学び |
|------|---------|-----------|-----------|
| フック手法 | ... | ... | ... |
| 情報密度 | ... | ... | ... |
| ヘッジ表現 | ... | ... | ... |
```

### Step 7: 一時ファイルのクリーンアップ

分析完了後、以下の一時ファイルを削除する:
- `ref_<ID>.ja.vtt` — ダウンロードしたVTTファイル
- `ref_<ID>.txt` — クリーンアップ済みテキスト
- `yt_meta.json` — メタデータJSON

knowledge/references/ に保存したレポートが成果物。中間ファイルは残さない。

## 判断のガイドライン

- ナレーション全文の読み込みは必須。長い動画でも省略しない
- 自動字幕の誤認識は文脈から推定して補正する。ただし補正箇所が多い場合はその旨を記載する
- 分析は「この動画が良い理由」を言語化することが目的。評点をつけることが目的ではない
- テクニック抽出では抽象的な評価（「構成が良い」）ではなく、具体的な手法の記述（「冒頭で3つの問いを提示し、本論で順に回収する」）を優先する
- ヘッジ表現の分析は特に丁寧に。センシティブなテーマでの表現技法は高度なスキルであり、具体例を網羅的に拾う
- `--compare` なしでも、「自分の動画への応用」セクションで自分の制作への接続を必ず含める
- レポートは後から横断検索できるよう、frontmatterのtagsを適切に設定する
